# 開發計畫書 v1.0

## 專案名稱
Manga Translator Studio（漫畫翻譯嵌字工具）

---

## 一、專案目標

打造一款「純本機桌面工具」，提供以下功能：

- 漫畫文字區偵測
- OCR 文字辨識
- 使用者自訂規則翻譯
- 支援 OpenAI / Gemini API（BYOK）
- 自動排版與嵌字
- 手動調整編輯
- 匯出成品與專案檔

---

## 二、產品定位

- 類型：桌面工具（Desktop Application）
- 模式：本機執行 + 使用者自備 API key
- 核心價值：
  - 提升漫畫翻譯效率
  - 減少 Photoshop / 手動嵌字成本
  - 保持術語一致性

---

## 三、目標使用者

- 同人翻譯組
- 漫畫漢化者
- 字幕組
- 個人翻譯者
- 輕小說翻譯者

---

## 四、技術架構（方案 A：manga_ocr_pipeline）

- UI：PySide6 / PyQt6
- OCR：`manga_ocr_pipeline`（RT-DETR-v2 文字框偵測 + manga-ocr-base 辨識）
- 圖像處理：OpenCV + Pillow
- 翻譯 API：OpenAI / Gemini
- 資料儲存：SQLite + JSON
- 金鑰管理：keyring

---

### OCR 技術路線補充（MVP）

- 偵測模型：`ogkalu/comic-text-and-bubble-detector`（RT-DETR-v2）
- 辨識模型：`kha-white/manga-ocr-base`
- 執行框架：`transformers + torch`
- 流程：先偵測文字框，再逐框 OCR，最後依閱讀方向排序輸出
- 備援策略：當 OCR runtime 不可用時，可使用 fixture fallback 以維持測試流程

## 八、風險與對策

| 風險 | 對策 |
|------|------|
| OCR 不準 | 提供手動修正 |
| 排版困難 | 先簡化規則 |
| API 失敗 | 重試機制 |
| 漫畫格式複雜 | 使用者可手動調整 |

---

## 九、MVP 範圍（重要）

只做：

- 單頁處理
- OCR
- 翻譯
- 基本嵌字
- 匯出 PNG

不做：

- 批次
- 插件
- 高級排版

---

## 十、成功指標

- 單頁處理 < 10 秒
- 使用者能完成一頁翻譯
- 翻譯流程無需外部工具
